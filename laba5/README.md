# Контрольні питання

## 1. Порівняння методів сортування

### **Сортування бульбашкою (Bubble Sort)**

**Переваги:**

* дуже простий для реалізації;
* легко зрозуміти принцип роботи;
* може завершитися раніше, якщо масив уже відсортований.

**Недоліки:**

* повільний: середній та гірший випадок — **O(n²)**;
* багато непотрібних порівнянь і перестановок;
* майже не використовується на практиці.

---

### **Сортування вибором (Selection Sort)**

**Переваги:**

* мінімальна кількість обмінів (лише **n − 1**);
* працює однаково для будь-якого масиву;
* проста реалізація.

**Недоліки:**

* завжди має час виконання **O(n²)**;
* не є стійким;
* не підходить для великих обсягів даних.

---

### **Сортування вставкою (Insertion Sort)**

**Переваги:**

* дуже ефективний на частково відсортованих масивах;
* найкращий випадок — **O(n)**;
* є стійким;
* добрий вибір для малих масивів.

**Недоліки:**

* гірший випадок — **O(n²)**;
* багато зсувів елементів при даних у зворотному порядку.

---

## 2. Оцінка кожного методу

| Алгоритм           | Час роботи            | Додаткова пам’ять | Стабільність |
| ------------------ | --------------------- | ----------------- | ------------ |
| **Bubble Sort**    | O(n²), найкращий O(n) | O(1)              | ✔ стійкий    |
| **Selection Sort** | завжди O(n²)          | O(1)              | ✘ не стійкий |
| **Insertion Sort** | O(n²), найкращий O(n) | O(1)              | ✔ стійкий    |

### **Додаткові критерії порівняння:**

* кількість порівнянь;
* кількість перестановок/зсувів;
* адаптивність (чи прискорюється на відсортованих масивах);
* складність реалізації;
* ефективність для малих або великих наборів даних.

---

## 3. Найкращий і найгірший випадки

### **Bubble Sort**

* **Найкращий:** масив відсортований → O(n)
* **Найгірший:** зворотний порядок → O(n²)

### **Selection Sort**

* **Найкращий:** O(n²)
* **Найгірший:** O(n²)
  *(Порядок не впливає на швидкість.)*

### **Insertion Sort**

* **Найкращий:** масив частково відсортований → O(n)
* **Найгірший:** зворотний порядок → O(n²)

### **Висновки:**

* Найкращий на частково впорядкованих даних → **Insertion Sort**
* Найстабільніший за часом (не міняється) → **Selection Sort**
* Найгірші випадки для Bubble та Insertion майже однакові → **O(n²)**

---

## 4. Пояснення позначень O(1), O(n), O(n²)

* **O(1)** — константна складність: час не залежить від кількості елементів.
* **O(n)** — лінійна складність: час роботи зростає пропорційно розміру масиву.
* **O(n²)** — квадратична складність: час збільшується пропорційно квадрату кількості елементів.
